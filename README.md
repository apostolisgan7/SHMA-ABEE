## Getting Started

### Requirements
- Node.js >= 18
- WordPress >= 6.0
- WooCommerce >= 8.0
- Advanced Custom Fields PRO

### Installation

```bash
cd wp-content/themes/Ruined
npm install
```

### Development

```bash
npm run dev
```

Starts Vite dev server at `http://localhost:5173`. WordPress detects dev mode automatically via the absence of `dist/.vite/manifest.json`.

### Production Build

```bash
npm run build
```

Outputs compiled assets to `dist/`. WordPress switches to manifest-based asset loading automatically.

## Development Workflow

### NPM Scripts

| Command | Description |
|---|---|
| `npm run dev` | Start Vite dev server (clears dist first) |
| `npm run build` | Production build (clears dist first) |
| `npm run preview` | Preview production build locally |

### How Dev/Prod Switching Works

`vite.php` checks if `dist/.vite/manifest.json` exists:
- **Dev** (manifest absent): Vite injects scripts via `wp_head`. Assets served from `localhost:5173`.
- **Production** (manifest present): WordPress enqueues fingerprinted files from `dist/` using the manifest.

### Global JS Object

Both modes expose `window.rv_globals`:

```js
rv_globals.ajaxurl  // WordPress admin-ajax.php URL
rv_globals.nonce    // rv_ajax_nonce — required for all theme AJAX requests
```

### Hero Video Loading Strategy

Videos in hero blocks are deferred to avoid blocking LCP:
- HTML has `preload="none"`, no `autoplay`, `poster` set to hero image URL
- `initHeroVideo()` calls `video.play()` only after `window.load` + 500ms
- YouTube/Vimeo: `src` in `data-src`, activated by JS after load
- Slow connections (`navigator.connection.saveData` or 2G/3G) skip video entirely

## User Roles & Auth System

### Account Types

| Type | WP Role | Auto-Login | Requires Approval |
|---|---|---|---|
| Individual Customer | `customer` (WC default) | Yes | No |
| Company (B2B) | `company` | No | Yes — admin must approve |
| Municipality | `municipality` | No | Yes — admin must approve |

### Pending Approval Flow
1. Company/municipality registers → meta `_sigma_account_status = 'pending'` set
2. Admin receives email with direct link to user edit page in wp-admin
3. Admin approves → clears pending status
4. Pending users attempting to log in receive a rejection message and are logged out

### Rate Limiting (transient-based, keyed by IP)

| Action | Limit | Window |
|---|---|---|
| Login | 5 attempts | 10 minutes |
| Register | 5 attempts | 10 minutes |
| VAT check | 10 checks | 5 minutes |

### VAT Validation
- **Companies:** validated in real-time against VIES EU VAT database
- **Municipalities:** format check only (9 Greek digits, no VIES)
- **Cache:** valid results cached 6h, invalid 1h (WordPress transients)

### Auth Files
```

├── auth/
│   ├── ajax-check-vat.php   # Real-time VAT check on field blur
│   ├── ajax-login.php       # Login AJAX handler

├── account-fields.php       # Extra fields in My Account > Account Details
├── account-roles.php        # Custom role definitions

```

## AJAX Endpoints

All endpoints available to both logged-in and guest users. All protected with nonces.

| Action | Nonce | Handler | File |
|---|---|---|---|
| `sigma_login` | `sigma-login` | `sigma_ajax_login()` | `auth/ajax-login.php` |
| `sigma_register` | `sigma-register` | `sigma_ajax_register()` | `auth/ajax-register.php` |
| `sigma_check_vat` | `sigma-register` | `sigma_ajax_check_vat()` | `auth/ajax-check-vat.php` |
| `ruined_update_cart_qty` | `ruined_cart_nonce` | `ruined_update_cart_qty()` | `woocommerce/configurations.php` |
| `rv_load_more_products` | `rv_ajax_nonce` | `rv_load_more_products()` | `woocommerce/load-more.php` |

All theme AJAX requests must include `rv_globals.nonce` (except auth endpoints which use their own nonces from form hidden fields).
